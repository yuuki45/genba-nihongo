# App Store 表記ドラフト（名前・サブタイトル・キーワード）

作成: 2026-06-08 / 対象: App Store Connect の「製品情報」をローカリゼーションごとに入力

## 方針

- ホーム画面のアイコン名は **Nihongo** のまま（短くアイコン下に収まる）
- App Store の表示名・サブタイトルは**言語ごとに別設定**できる。実習生は日本のApp Storeでも
  端末がインドネシア語なら id 版の表記が表示される → **id を主ターゲットとして最適化**
- ブランドは **Genba Nihongo（現場にほんご）**。ありふれた "Nihongo" 単体との差別化になる
- App Store の文字数上限: **名前 30字 / サブタイトル 30字**（全角も半角も1字）

## インドネシア語ロケール（id）← 主ターゲット

| 項目 | 案 | 文字数 |
|------|-----|:---:|
| **名前** | `Genba Nihongo: Bahasa Jepang` | 28 |
| **サブタイトル** | `Frasa pabrik, perawatan & JLPT` | 30 |

- 別サブタイトル案: `Frasa kerja, perawatan & JLPT`（29字／"kerja"=仕事でより広く）
- **キーワード欄**（100字・非表示・スペースなしのカンマ区切り。名前/サブタイトルの語は入れない）:
  ```
  belajar,kanji,N3,N2,N4,N5,kaigo,magang,kerja,kosakata,hiragana,katakana,kamus,TKI,offline,ujian
  ```
  （95字。名前/サブタイトルにある bahasa/jepang/nihongo/genba/frasa/pabrik/perawatan/JLPT は重複を避けて除外）
  - インドネシア人は漢字では検索しない → すべてローマ字・インドネシア語。「kanji」「hiragana」のように
    日本語の概念もローマ字で打つ前提（"辞書"ではなく "kamus"）
  - ⚠️ **ASCで必ず「インドネシア語」ローカリゼーションを追加すること**。なければ主言語（日本語＝漢字）の
    キーワードがフォールバックされ、インドネシア語端末のユーザーに検索ヒットしなくなる

> インドネシア人実習生の多くは端末がインドネシア語なので、彼らに効くのは id 版。
> ja 版は日本語話者（雇用側・支援者など）向け、および主言語フォールバック用。

## 補足

- 「名前」は検索順位への影響が最も大きい → ブランド＋最重要キーワード（Bahasa Jepang / 日本語）
- 「サブタイトル」は対象と中身（職種＋JLPT）を凝縮
- 名前・サブタイトル・キーワードに**同じ語を重複させない**（インデックスの無駄。例: 名前に Bahasa Jepang を入れたらキーワードからは外してよい）
- アイコン名（CFBundleDisplayName / android:label）は現状 `Nihongo` のまま変更なし

### キーワード欄の入力ルール（重要）
- **スペースを入れない**（`a,b,c`。`a, b, c` はスペース分の文字数を無駄にする）
- 単数/複数や語の組み合わせはApp Storeが自動で処理 → **複合語を丸ごと入れず、単語に分解**して入れる
- アプリ名・サブタイトル・カテゴリ名・他社アプリ名は入れない（名前/サブタイトルは既に検索対象）
- 100字に収まる範囲で、検索されそうな語を詰める
